# BiliQueue

BiliQueue 是一个本地运行的 B 站直播排队工具。程序提供网页控制台与 OBS 浏览器横条，通过直播间弹幕管理排队、取消排队和礼物插队。

> 当前版本：v0.1.6
>
> 当前处于测试阶段。B 站直播接口可能发生调整，建议使用 Release 中的最新版本。

## 功能

- 使用直播间号连接 B 站直播弹幕
- B 站扫码登录，登录信息仅保存在本机项目目录
- 通过弹幕指令加入或取消排队
- 当前用户、等待队列和说明区域独立显示
- 三个区域宽度及横条统一高度可调整
- 单行、双行队列及横向滚动
- 普通弹幕用户头像获取与本机缓存
- 按电池门槛触发礼物插队
- 礼物优先区可选择按单次礼物价值排序
- 手动新增、删除、拖动排序和移动到队尾
- 当天队列异常退出恢复
- 配置导入、导出与导入前自动备份

## 下载与运行

请从仓库的 **Releases** 页面下载：

```text
BiliQueue-v0.1.6-windows-test.zip
```

解压后双击：

```text
启动-BiliQueue.cmd
```

控制台地址：

```text
http://127.0.0.1:18303/control
```

OBS 浏览器来源地址：

```text
http://127.0.0.1:18303/overlay
```

OBS 浏览器来源的宽度和高度，请按照控制台底部显示的建议值填写。

## 基本使用

1. 启动 BiliQueue。
2. 首次使用时，在控制台点击“扫码登录”。
3. 填写直播间号并连接。
4. 观众发送 `排队` 加入队列，发送 `取消排队` 退出队列。
5. 在 OBS 中添加浏览器来源并填写横条地址。
6. 根据直播需求调整区域尺寸、文字样式、滚动方式和礼物规则。

弹幕指令、空态文字和右侧说明内容均可在控制台修改。

## 礼物插队

礼物插队按单次付费礼物价值判断，低于门槛的多次礼物不会累计。

关闭“按礼物价值排序”时，新触发插队的用户会追加到现有礼物优先区末尾：

```text
当前A → D（先送礼）→ C（后送礼）→ 普通队列
```

开启后，礼物优先区按最近一次达到门槛的单次礼物价值从高到低排列。

免费礼物默认不参与电池门槛计算。

## 数据位置与升级

所有运行数据均保存在程序所在目录的 `data` 文件夹中：

```text
data/
├── auth.json
├── config.json
├── queue-today.json
├── avatars/
└── backups/
```

其中 `auth.json` 含 B 站登录 Cookie，请勿分享或提交到 GitHub。

升级方式：

- 在原目录覆盖新版 EXE，可以继续使用原登录状态和配置。
- 解压到新目录时，将旧版 `data` 文件夹复制到新目录。

仓库已通过 `.gitignore` 排除用户数据、缓存、构建产物和登录信息。

## 配置导入与导出

控制台支持导出和导入配置。

导出的配置包括直播间、指令、礼物规则、OBS 布局和文字样式，不包括：

- B 站登录 Cookie
- 当前队列
- 当天队列快照
- 头像缓存
- 最近弹幕或礼物记录

导入前，当前配置会自动备份到 `data/backups`。

## 源码构建

要求：

- Go 1.23 或更高版本
- 无外部 Go 模块依赖

运行测试：

```bash
go test ./...
go test -race ./...
go vet ./...
```

Windows AMD64 构建：

```bash
GOOS=windows GOARCH=amd64 CGO_ENABLED=0 go build -trimpath -ldflags="-s -w" -o BiliQueue-windows-amd64.exe .
```

## 项目提交方式

仓库只提交已经确认的版本。功能开发、试验和测试包会先在本地工作目录完成；确认版本内容后，再统一提交源码并发布 Release。未确认的中间修改不会直接写入仓库。

## 隐私说明

BiliQueue 为本地程序。账号 Cookie、配置、队列和头像缓存保存在本机项目目录。使用源码或发布包时，请自行保护 `data` 文件夹中的敏感内容。
