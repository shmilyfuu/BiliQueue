# BiliQueue v0.1.12

本版本为 v0.1.11 之后的确认发布版。v0.1.12-test1 至 v0.1.12-test8 为本地测试迭代，不单独发布 Release。

## 新增与优化

- 新增端口配置：端口写入配置，下次启动沿用。
- Windows 启动时端口被占用会弹出输入框，可改用新的本地端口或地址。
- 控制台增加本机服务地址修改入口，修改成功后自动跳转。
- 托盘菜单增加复制控制台地址、修改端口、打开简易控制页。
- 新增单实例运行，重复启动时提示已有 BiliQueue 正在运行。
- 新增 `/mini-control` 简易控制页，用于队列管理和横条文字修改。
- 新增“是否开启排队”开关。关闭后弹幕排队和礼物插队不触发，右侧说明区域只显示说明内容。
- 新增主播清空队列弹幕指令，默认 `清空队列`，可在控制台自定义。
- 修正双行队列第一行被拉开的问题。
- 新增队列用户间距、第二行每页最多人数。
- 双行第二行分页改为上下切换，并按实际宽度判断每页显示数量。
- 新增淡入淡出模式，分页和淡入淡出支持停留时间、切换耗时设置。
- 控制台参数区支持折叠，条幅大小与样式、队列调整与样式、文字区域分为独立设置组。
- 控制台顶部标题只显示主版本号，例如 `BiliQueue v0.1.12`。
- 控制台文案统一为浏览器源描述，减少特定直播软件名称。
- 优化数值输入框样式，数值框右侧增加上下微调按钮。

## 修复

- 托盘退出改为先保存配置和队列，再立即关闭本地服务，减少退出后进程短暂残留。
- 端口占用和修改端口弹窗统一为自绘窗口，优化标题、提示、输入框布局和按钮。
- 启动时检测到端口被占用，点击“取消”后直接取消启动，不再弹出额外错误框。
- 检测到已有 BiliQueue 正在运行时，只显示提示，不再自动打开控制台页面。
- 修复托盘“修改端口”窗口可能未响应的问题，并避免重复打开多个修改端口窗口。
- 修复分页滑动切换后内容可能消失的问题。
- 修复切换滚动方式后部分效果需要刷新才生效的问题。
- 修复双行第二行超出队列区域后可能整行消失的问题。
- 修复“第二行每页最多人数”参数不生效的问题。
- 调整恢复默认按钮，使其尺寸和文字大小与收起 / 展开按钮一致。

## 下载

- `BiliQueue-v0.1.12-windows.zip`：Windows AMD64 包
- `BiliQueue-v0.1.12-source.zip`：完整源码

## 升级提示

需要保留登录状态和现有设置时，请保留旧版的 `data` 文件夹。需要保留自定义字体时，请同时保留 `fonts` 文件夹。`data/auth.json` 含 B 站登录 Cookie，请勿分享。

## BiliQueue v0.1.11

本版本为 v0.1.10 之后的确认发布版。v0.1.11-test1 至 v0.1.11-test6 为本地测试迭代，不单独发布 Release。

### 新增与优化

- Windows 版改为无控制台窗口程序。
- 支持直接双击 `BiliQueue-windows-amd64.exe` 静默启动到系统托盘。
- 将 BiliQueue 图标写入 exe 文件本身，资源管理器中会显示应用图标。
- 增加系统托盘右键菜单：打开控制台、打开横条、复制浏览器源地址、清空队列、打开数据文件夹、打开日志文件和退出。
- 增加本地日志文件：`data/logs/biliqueue.log`。
- 将横条文字内容编辑区移动到控制台前部。
- 编辑横条文字时不再实时同步，需点击“应用文字到横条”后才保存并同步到横条。
- 将启动脚本改为 `start-BiliQueue.cmd`，避免 Windows / zip / GitHub Actions 间的文件名编码问题。
- Windows 测试包说明文件统一为 `README.md`。

### 修复

- 修复托盘菜单第二次右键可能无法弹出的问题。
- 修复托盘菜单“退出”可能无法结束进程的问题。
- 修复 GUI 模式下日志可能没有输出的问题。
- 降低重复打开控制台或横条时页面偶发卡在加载中的概率。
- 统一程序数据目录为 exe 所在目录，直接双击 exe 与 cmd 启动读取同一份 `data`、`fonts`、`assets`。

### 下载

- `BiliQueue-v0.1.11-windows-test.zip`：Windows AMD64 测试包
- `BiliQueue-v0.1.11-source.zip`：完整源码

### 升级提示

需要保留登录状态和现有设置时，请保留旧版的 `data` 文件夹。需要保留自定义字体时，请同时保留 `fonts` 文件夹。`data/auth.json` 含 B 站登录 Cookie，请勿分享。
