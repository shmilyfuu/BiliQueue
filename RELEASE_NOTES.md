# BiliQueue v0.1.14

本版本为 v0.1.13 之后的控制台与插件展示优化版，重点重构控制台操作界面，并修复双行队列、连续滚动和礼物用户展示在动态更新时的布局问题。

## 新增与优化

- 控制台左右列改为等宽布局，主要卡片统一使用 18px 内边界和 45px 单行控件高度。
- 顶部预览区重新排列浏览器源地址、尺寸建议、操作按钮和连接状态。
- 队列管理改为拖拽排序；支持点选队列用户，并让模拟礼物插队作用于选中用户。
- 模拟礼物插队新增礼物价值输入框。
- 重构“文字编辑区”“排队与礼物规则”“大小与样式”“队列调整与样式”和“文字区域样式”布局。
- 文字编辑区和三个区域文字样式中的文字内容双向同步，并可在对应区域就地应用或放弃修改。
- 新增队列空态文字对齐，以及当前、队列和说明区域内部多行文字行间距控制。
- 控制台会在浏览器本地记住每个卡片的展开或收起状态。
- 页面右下角新增回到顶部按钮。
- 配置写入改为串行执行，降低自动保存与恢复默认、导入配置或应用文字相互覆盖的风险。
- 补充控制台和简易控制页异步操作的错误提示，并为复制地址增加兼容回退。

## 插件显示优化

- 调整礼物用户的序号、礼物图标、头像和用户名间距，使其统一受“队列区域头像文字间距”控制。
- 礼物用户高亮改为由左向右透明衰减的填充与描边，普通用户和礼物用户保持同一内容起点。
- 连续滚动模式改为与分页滑动、淡入淡出相同的固定列布局，单双行保持一致。
- 连续滚动更新队列时一次性生成完整循环轨道，并保留当前滚动位置，减少新增或移除用户时的瞬间位移。
- 优化分页滑动在单双行模式下的上下边缘渐隐位置。

## 修复

- 修复控制台移除或新增用户后，插件双行队列第二行可能布局异常的问题。
- 修复只有控制台模拟排队更新时第二行不能及时恢复布局的问题。
- 修复连续滚动超过同屏人数后，新增或移除用户会短暂跳回起点的问题。
- 修复真实礼物图标与模拟礼物占位图标垂直对齐不一致的问题。
- 修复文字区域颜色面板在页面底部关闭后可能导致页面向上滚动的问题。
- 修复说明区域文字行间距滑块没有作用于内部多行文字的问题。
- 修复文字区域下拉菜单可能出现白色背景和白色文字的问题。
- 修复队列管理高度会随右侧规则卡片收起或展开而改变的问题。

# BiliQueue v0.1.13

本版本为 v0.1.12 之后的确认发布版，重点修复横条双行布局和分页滑动显示细节，并增加更灵活的区域开关与本地测试启动参数。

## 新增与优化

- “大小与样式”中新增“启用当前区域”和“启用说明区域”开关；关闭后对应区域不显示，也不占横条宽度。
- “统一高度”最小值改为 0，恢复默认值改为 50。
- 分页滑动改为上下边缘渐隐蒙版效果，保留向上滑动但不再使用整页淡入淡出。
- 双行分页滑动时，第二行使用独立渐隐蒙版，使渐隐位置贴合实际滑动行。
- 新增 `-fonts` 启动参数，便于本地测试时将数据目录和字体目录分别指向临时位置。

## 修复

- 修复横条从单行首次切换到双行时，第二行可能使用未稳定宽度计算布局而显示异常的问题。
- 修复分页滑动伪循环回到第一页时可能闪烁的问题。
- 修复分页滑动只有一页内容时仍套用渐隐蒙版的问题。
- 修正 Windows 剪贴板写入逻辑中的 unsafe.Pointer vet 提示。

## 下载

- `BiliQueue-v0.1.13-windows.zip`：Windows AMD64 包
- `BiliQueue-v0.1.13-source.zip`：完整源码

## 升级提示

需要保留登录状态和现有设置时，请保留旧版的 `data` 文件夹。需要保留自定义字体时，请同时保留 `fonts` 文件夹。`data/auth.json` 含 B 站登录 Cookie，请勿分享。

# BiliQueue v0.1.12

本版本为 v0.1.11 之后的确认发布版。v0.1.12-test1 至 v0.1.12-test25 为本地测试迭代，不单独发布 Release。

## 新增与优化

- 新增端口配置：端口写入配置，下次启动沿用。
- Windows 启动时端口被占用会弹出输入框，可改用新的本地端口或地址。
- 控制台增加本机服务地址修改入口，修改成功后自动跳转。
- 托盘菜单增加复制控制台地址、修改端口、打开简易控制页。
- 新增单实例运行，重复启动时提示已有 BiliQueue 正在运行。
- 新增 `/mini-control` 简易控制页，用于队列管理。
- 新增“是否开启排队”开关。关闭后弹幕排队和礼物插队不触发，右侧说明区域只显示说明内容。
- 新增主播清空队列弹幕指令，默认 `清空队列`，可在控制台自定义。
- 新增主播下一位弹幕指令，默认 `下一位`，可在控制台自定义。
- 登录状态存在且已保存直播间号时，启动后自动连接弹幕。
- 连续滚动改为固定格子长条逻辑，用户名省略长度按统一格子宽度计算。
- 当前区域“当前”标记改为浮在头像左上角的小卡片，并支持位置偏移。
- 修正双行队列第一行被拉开的问题。
- 新增每行列数、列间距和双行显示设置。
- 单行和双行分页改为上下切换，固定列布局按每行列数和列间距计算。
- 新增淡入淡出模式；分页滑动加入渐入渐隐，分页和淡入淡出支持停留时间、切换耗时设置。
- 控制台参数区支持折叠，条幅大小与样式、队列调整与样式、文字区域分为独立设置组。
- 控制台顶部标题只显示主版本号，例如 `BiliQueue v0.1.12`。
- 控制台文案统一为浏览器源描述，减少特定直播软件名称。
- 优化数值输入框样式，数值框右侧增加上下微调按钮。

## 修复

- 托盘退出改为先保存配置和队列，再立即关闭本地服务，减少退出后进程短暂残留。
- 端口占用和修改端口弹窗统一为自绘窗口，优化标题、提示、输入框布局和按钮。
- 启动时检测到端口被占用，点击“取消”后直接取消启动，不再弹出额外错误框。
- 检测到已有 BiliQueue 正在运行时，只显示提示，不再自动打开控制台页面。
- 修复托盘“修改端口”窗口可能未响应的问题，并避免重复打开多个修改端口窗口。
- 修复分页滑动切换后内容可能消失的问题。
- 修复切换滚动方式后部分效果需要刷新才生效的问题。
- 修复双行第二行超出队列区域后可能整行消失的问题。
- 修复“第二行每页最多人数”参数不生效的问题。
- 调整恢复默认按钮，使其尺寸和文字大小与收起 / 展开按钮一致。

### 新增与优化

- Windows 版改为无控制台窗口程序。
- 支持直接双击 `BiliQueue-windows-amd64.exe` 静默启动到系统托盘。
- 将 BiliQueue 图标写入 exe 文件本身，资源管理器中会显示应用图标。

- 修复托盘菜单第二次右键可能无法弹出的问题。

- 降低重复打开控制台或横条时页面偶发卡在加载中的概率。
- 统一程序数据目录为 exe 所在目录，直接双击 exe 与 cmd 启动读取同一份 `data`、`fonts`、`assets`。
